1. **Открой новый чат** — не продолжай старую сессию с другой задачей

## ШАГ 0 — Переформулировать запрос и выбрать маршрут

**Input:** запрос пользователя
**Output:** артефакт `TASK_PROFILE`

### 0.1 Заполнить все 7 элементов

```
1. ОБЪЕКТ        — что именно (не «пылесосы», а «роботы-пылесосы» — разные рынки)
2. ГЕОГРАФИЯ     — рынок РФ (если не указано иное)
3. МЕТРИКИ       — в деньгах (млрд ₽) И в штуках — ОБА, ВСЕГДА раздельно
4. ГОРИЗОНТ      — факт [годы] + прогноз до [год]
5. СЕГМЕНТЫ      — ценовые границы конкретными цифрами (не «бюджет/премиум»)
6. ИГРОКИ        — доли рынка (не просто «кто есть»)
7. ДОП. КОНТЕКСТ — драйверы роста + барьеры + JTBD если известны
```

### 0.2 Decision Tree — выбрать ветки

```
▸ Consumer B2C (физический товар, маркетплейсы / розница):
  ОБЯЗАТЕЛЬНО: A (отчёты) + B (маркетплейсы) + G (top-down)
  По ситуации: C (Вордстат) + H (бридж) если данных мало

▸ B2B / инфраструктура / промышленность:
  ОБЯЗАТЕЛЬНО: A (отчёты) + D (реестры) + E (таможня)
  МП не релевантны

▸ Niche consumer / IoT / нет аналогов в РФ:
  ОБЯЗАТЕЛЬНО: A (отчёты) + H (бридж)
  По ситуации: B + C

▸ Сервис / подписка / SaaS:
  ОБЯЗАТЕЛЬНО: A (отчёты) + G (top-down аудитория) + D (конкурентная выручка)

ЛОГИКА ДОСТАТОЧНОСТИ:
  ≥3 метода + интервал <30% → дополнительные ветки опциональны
  <3 методов или интервал >30% → добавить ветку обязательно
  Интервал >60% → СТОП, сообщить пользователю до продолжения
```

### 0.3 Зафиксировать профиль

Кто юзает:        [пользователь]

```

### Ветка A: Чужие отчёты (ОБЯЗАТЕЛЬНАЯ для всех типов)

**Лимит:** max 5 запросов

```

**КРИТИЧНО:** 5 статей цитируют ITResearch → это 1 источник × 5 зеркал.

**SELF-CHECK перед записью в RAW_DATA:**
```
□ Реально перешёл по каждой ссылке (не принял заголовок за содержание)?
□ Все отчёты — независимые агентства, не зеркала одного первоисточника?
□ Дата каждого отчёта известна и указана?
□ Statista по РФ после 2022 — не использован как основной?
□ Цифры разделены на деньги и штуки отдельно?
□ Есть ≥1 источника не из отраслевых изданий?
Хоть один □ → конкретное исправление

### Ветка B: Маркетплейсы (ОБЯЗАТЕЛЬНАЯ для Consumer B2C)

Гео-блок: WB/Ozon API недоступны с этого IP.

**Запросить у пользователя:**
```
Нужны данные маркетплейсов. Выбери удобное:
1. data.ozon.ru → категория [X] → скопируй таблицу GMV
2. ozon.ru → поиск «[запрос]» → скриншот первых 20 товаров с ценами
3. wb.ru → поиск «[запрос]» → скриншот
```

Если пользователь не предоставил данные → «Ветка B: данные МП не получены» → следующая ветка. Не ждать бесконечно.

**Когда данные получены:**
```
Метод 1 (через GMV):
  GMV категории ÷ 0.69 = весь онлайн ÷ 0.19 = весь рынок

Метод 2 (через отзывы — если нет GMV):
  Отзывы за 30-90 дней × 30-50 (электроника) = продажи шт
  × средняя цена = выручка МП
  → применить формулу выше
```

> ⚠️ **МП = ~30% рынка (данные отраслевых аналитиков):**
> Доли по МП ≠ доли по всему рынку. Бренд с 24% на Ozon → ~10-15% рынка.
> - Объём: МП × ~3.3 = оценка рынка ✅
> - Доли игроков с МП: только с пометкой «МП-доля, ~30% рынка» ⚠️
> - Для реальных долей → Марвел Дистрибуция, GfK, пресс-релизы ритейлеров
>
> ⚠️ Загрязнение Ozon: дубли дешёвых товаров под высокой ценой. Данные >40к ненадёжны.

**SELF-CHECK:**
```
□ Пометил «МП-доля, ~30% рынка» — не рыночная?
□ Отзывы за 30-90 дней (не total за всё время)?
□ Дубли, мусорные листинги, аксессуары вычтены?
□ Данные разделены на деньги и штуки?
Хоть один □ → пересчитать / написать «Ветка B: нет данных»
```

**После ветки — записать в протокол:**
```
Ветка B: [ПОЛУЧЕНО / МП-данные не получены от пользователя]
  МП GMV: [X млрд ₽] → × ~3.3 = ~[Y млрд ₽] рынка ⚠️МП-доля
  Штуки: [Z млн шт]
```

### Ветка C: Яндекс Вордстат (прокси спроса)

API-ключ: `<YOUR_WORDSTAT_TOKEN>`

```
Запросы/мес × 1.7 (mobile) × 3-8% (conversion) × средняя цена
= оценка месячной выручки × 12 = годовой объём

Ограничения: прокси спроса, не продаж. Сезонность видна (2 года истории).

**SELF-CHECK:**

□ Только прокси, не основная цифра?
□ Не взял пиковый месяц за среднее (проверил сезонность)?
□ Conversion rate 3-8% с обоснованием для конкретной категории?
□ Помечен «proxy спроса»?
Хоть один □ → скорректируй параметр → пересчитай
```

**После ветки — записать в протокол:**
```
Ветка C: [ПОЛУЧЕНО / НЕ ВЫПОЛНЯЛАСЬ]

□ Данные выручки за последний отчётный год?
□ Доля топ-10 — есть источник, не из головы?
Хоть один □ → уточни ОКВЭД / источник доли

```

**После ветки — записать в протокол:**
```
Ветка D: [ПОЛУЧЕНО / НЕ НАЙДЕНО]
  Сумма топ-10 выручки: [X млрд ₽] ÷ [Y% доля] = ~[Z млрд ₽] рынка
```

### Ветка E: Таможенные данные (физические импортные товары)

Источник: `customs.gov.ru`, HS-код через `search.customs.ru` или классификатор ТН ВЭД

```
Импорт по HS-коду ≈ нижняя граница объёма рынка
(если нет значимого отечественного производства)
```

> ⚠️ Ограничения:
> - После 2022: серый импорт и параллельный ввоз — цифра занижена
> - Не учитывает отечественное производство
> - Одна позиция HS может включать несколько разных продуктов
> → Нижняя граница, не точная оценка

**SELF-CHECK:**
```
□ HS-код точно соответствует именно этому продукту?
□ Учёл что после 2022 серый импорт значим — цифра занижена?
□ Есть отечественное производство? Если да — таможня ≠ весь рынок
□ Показан как «нижняя граница», не как точная оценка?
Хоть один □ → скорректируй интерпретацию
```

**После ветки — записать в протокол:**
```
Ветка E: [ПОЛУЧЕНО / НЕ НАЙДЕНО]
  Импорт по HS [код]: [X тыс. шт] / [Y млрд ₽] — нижняя граница ⚠️
```

### Ветка F: Поведенческие (UX) данные

⚠️ **Самый галлюциногенный раздел.** Модель часто генерирует правдоподобные, но неверные числа.
Пример: скилл выдал «90% не используют ежедневно» — реальные данные М.Видео: 48% используют каждый день.

**Правило: нет источника исследования → цифра не включается в отчёт вообще.**
Не помечается, не оговаривается — просто отсутствует.

Надёжные источники: М.Видео Research, GfK Россия, Mediascope (TNS), ВЦИОМ / ФОМ / Левада

**SELF-CHECK:**
```
□ У каждой цифры — ссылка на конкретное исследование с выборкой?
□ Не сгенерировал «правдоподобную» цифру из головы?
Хоть один □ → цифру убрать
Написать: «поведенческих данных с источником не найдено» — это честнее
```

**После ветки — записать в протокол:**
```
Ветка F: [ПОЛУЧЕНО с источниками / ДАННЫХ НЕТ]
```

### Ветка G: Top-down по аудитории (ОБЯЗАТЕЛЬНАЯ для Consumer B2C)

```
Источники:
- Росстат (rosstat.gov.ru) — население, доходы, регионы
- DBnomics (db.nomics.world/ROSSTAT) — API
- sberindex.ru — динамика потребительских расходов по данным крупнейшего банка РФ

5-факторная локализация (НЕ просто «население РФ / мировое»):
1. Зрелость рынка (инфраструктура, логистика готовы?)
2. Регуляторика (ГОСТы, субсидии, запреты)

### 3.1 Обратная подстановка — минимум 3 угла

```
КАЖДАЯ КЛЮЧЕВАЯ ЦИФРА = 3 НЕЗАВИСИМЫЕ ПРОВЕРКИ:
1. Прямой источник (отчёт / данные из ветки)
2. Обратный расчёт (через другую метрику)
3. Здравый смысл («это вообще реалистично для РФ?»)

Пример для рынка 40 млрд ₽, 2 млн шт:

Угол 1 (через аудиторию):
  50М домохозяйств × 8% пенетрация = 4М владельцев
  Цикл замены 5 лет → 800К покупок/год × 20К ₽ = 16 млрд
  НЕ БЬЁТСЯ с 40 млрд → вывод: пенетрация выше или есть B2B

Угол 2 (через конкурентов):
  Топ-3 = 60% = 24 млрд → проверить выручку через Rusprofile

Угол 3 (через таможню):
  Импорт по HS = X тыс. шт → сравнить с 2 млн шт
```

### 3.2 Аналитические ловушки (проверить всегда, не пропускать)

```

⚠️ Статистика внутри статистики:
   Одна цифра уже включает другую. Спроси: «эта цифра — часть предыдущей?»

⚠️ МП-доля ≠ рыночная доля:

```
